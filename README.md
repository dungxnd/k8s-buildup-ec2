# Kubernetes Cluster on EC2 Spot Instances

Bootstraps a 3-node Kubernetes cluster (1 master + 2 workers) on AWS EC2 spot instances, then deploys the [banking demo app](https://github.com/dungxnd/banking-demo-revamp) via its self-contained Helm chart. Zero-cost HTTPS available with DuckDNS + Let's Encrypt HTTP-01 challenge.

## Quick Start

```bash
# Edit terraform/terraform.tfvars with your key_name
cp terraform/terraform.tfvars.example terraform/terraform.tfvars

# Full pipeline — single command (HTTP-only, accessible via EC2 public IP)
make all

# Or with DuckDNS HTTPS enabled:
#   make infra
#   ansible-playbook -i ansible/inventory.ini ansible/site.yml --tags banking-demo \
#     -e 'banking_demo.duckdns.enabled=true' \
#     -e 'banking_demo.duckdns.domain=mybank.duckdns.org' \
#     -e duckdns_token=your-token

# Clean up
make destroy
```

## Architecture

```
k8s-buildup-ec2                        banking-demo-revamp
─────────────────                      ──────────────────
Terraform → 3 EC2 + SG                 Source code (Go + React)
Ansible   → OS + kubeadm + CNI         Helm chart (self-contained)
          → Clone repo                 CI (build + push images only)
          → DuckDNS A-record + cron
          → helm install

Chart deploys: Caddy (hostNetwork) → Kong (ClusterIP) → 4 services → Postgres/Redis/NATS
              with NetworkPolicies, shared-env ConfigMap, K8s Secrets

Giao tiếp duy nhất: kubeconfig + helm install --values.
Không coupling code. banking-demo là Helm chart self-contained.
```

## Manual Workflow (without Make)

```bash
# 1. Provision infrastructure
cd terraform
terraform init
terraform apply -auto-approve

# 2. Bootstrap cluster
ansible-playbook -i ansible/inventory.ini ansible/site.yml --tags os,k8s,cni,verify

# 3. Deploy app (HTTP-only, no DuckDNS)
ansible-playbook -i ansible/inventory.ini ansible/site.yml --tags banking-demo

# 3b. Deploy app with DuckDNS HTTPS (adds DuckDNS A-record + cron + caddy.tls in chart)
ansible-playbook -i ansible/inventory.ini ansible/site.yml --tags banking-demo \
  -e 'banking_demo.duckdns.enabled=true' \
  -e 'banking_demo.duckdns.domain=mybank.duckdns.org' \
  -e duckdns_token=your-token

# Clean up
cd terraform && terraform destroy
```

## Zero-Cost TLS Design

No Elastic IPs, no AWS Load Balancer. Every `terraform apply` gives new public IPs:

1. **Terraform** provisions 3 EC2 instances with randomly assigned public IPs.
2. **Ansible** updates the DuckDNS A-record immediately AND installs a `*/5 * * * *` cron job that keeps it in sync (spot instances can restart with new IPs).
3. **Caddy** (deployed by the banking-demo Helm chart as a Deployment with `hostNetwork: true`, pinned to the control-plane node) performs Let's Encrypt HTTP-01 challenge on port 80 — no API token needed in the chart.
4. **Caddy** reverse-proxies to `kong.banking.svc.cluster.local:8000` via CoreDNS. Everything behind Caddy is ClusterIP.

With DuckDNS disabled (`banking_demo.duckdns.enabled: false`, the default), the cluster runs HTTP-only on port 80 via Caddy. The Helm chart deploys Caddy regardless — TLS is just on/off via `caddy.tls.enabled`.

## Security Group Ports

| Port    | Protocol | Purpose                                    | Source     |
| ------- | -------- | ------------------------------------------ | ---------- |
| 22      | TCP      | SSH (Ansible)                              | admin_cidr |
| 80      | TCP      | HTTP (Caddy hostNetwork, Let's Encrypt)    | 0.0.0.0/0  |
| 443     | TCP      | HTTPS (Caddy hostNetwork, TLS termination) | 0.0.0.0/0  |
| 6443    | TCP      | Kube-apiserver                             | admin_cidr |
| 10250   | TCP      | Kubelet API                                | admin_cidr |
| 0-65535 | All      | Node-to-node (self-referencing)            | Self       |
| 0-65535 | All      | Outbound (egress)                          | 0.0.0.0/0  |

## CNI

**Cilium** is the default CNI, installed automatically via the `cni` role using the Cilium CLI. Cilium uses eBPF and does not require any extra firewall ports — node-to-node traffic flows through the self-referencing ingress rule.
