# Kubernetes Cluster on EC2 Spot Instances

Bootstraps a 3-node Kubernetes cluster (1 master + 2 workers) on AWS EC2 spot instances using Terraform for infrastructure and Ansible for OS preparation.

## Architecture

```
┌──────────────────────────────────────────┐
│  Terraform (infrastructure/)             │
│  ├── Default VPC (auto-discovered)       │
│  ├── Security Group (K8s ports)          │
│  ├── 3× EC2 Spot Instances (t3a.medium) │
│  │   └── Ubuntu 26.04 LTS               │
│  └── Auto-generates inventory.ini        │
└──────────────────┬───────────────────────┘
                   │
┌──────────────────▼───────────────────────┐
│  Ansible (ansible/)                      │
│  ├── Disable swap                        │
│  ├── Kernel modules (overlay, br_netfilter) │
│  ├── Sysctl (bridge-nf-call-iptables, ip_forward) │
│  ├── containerd (SystemdCgroup=true)     │
│  └── kubelet, kubeadm, kubectl (hold)    │
└──────────────────┬───────────────────────┘
                   │
┌──────────────────▼───────────────────────┐
│  Manual Bootstrap (kubeadm)              │
│  ├── kubeadm init (master)               │
│  ├── CNI: Cilium (default) or Calico     │
│  └── kubeadm join (workers)              │
└──────────────────────────────────────────┘
```

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/downloads) >= 1.5
- [Ansible](https://docs.ansible.com/ansible/latest/installation_guide/intro_installation.html)
- AWS credentials configured (`~/.aws/credentials` or `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY` env vars)
- An SSH key pair created in AWS EC2 (`YourKey` — or replace with yours)
- Ansible collections:
  ```bash
  ansible-galaxy collection install ansible.posix community.general
  ```

## Setup Steps

### 1. Configure

```bash
cp terraform/terraform.tfvars.example terraform/terraform.tfvars
```

Edit `terraform/terraform.tfvars`:
```hcl
key_name             = "YourKey"
ssh_private_key_path = "~/.ssh/YourKey.pem"
```

Only `key_name` is required. Everything else (VPC, subnet, AMI) is auto-discovered.

### 2. Provision Infrastructure

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

After apply, `ansible/inventory.ini` is generated automatically with real instance IPs.

### 3. Prepare OS

```bash
ansible-playbook -i ansible/inventory.ini ansible/setup.yml
```

### 4. Verify

```bash
ansible all -i ansible/inventory.ini -m ping
ansible all -i ansible/inventory.ini -a "kubeadm version"
ansible all -i ansible/inventory.ini -a "containerd --version"
```

## Security Group Ports

| Port      | Protocol | Purpose                | Scope   |
|-----------|----------|------------------------|---------|
| 22        | TCP      | SSH                    | All     |
| 6443      | TCP      | Kube-apiserver         | All     |
| 10250     | TCP      | Kubelet API            | All     |
| 30000-32767 | TCP   | NodePort Services      | All     |
| 179       | TCP      | Calico BGP (optional)  | All     |
| 4789      | UDP      | Calico VXLAN (optional)| All     |

## CNI

**Cilium** is the default CNI, installed via the Cilium CLI during manual bootstrapping.
Cilium uses eBPF and does not require any extra firewall ports beyond the self-referencing
ingress rule (node-to-node traffic is allowed on all ports).

**Calico** is available as an alternative — see [MANUAL_BOOTSTRAPPING.md](MANUAL_BOOTSTRAPPING.md#alternative-calico)
for installation instructions.

| Node count        | 3                  |

## Clean Up

```bash
cd terraform && terraform destroy
```

## Known Warning

A deprecation warning about `network_interface` on `aws_instance` is cosmetic — it originates from the provider internally reading a computed attribute. No functional impact. The top-level `subnet_id`/`vpc_security_group_ids` arguments remain current and supported by AWS.
